# HidenCloud 自动续期机器人 (HidenCloud Auto Renew)

本项目是一个高度自动化、智能化的 HidenCloud 服务器续期解决方案。它将 **Python (DrissionPage)** 的强大浏览器控制能力与 **GitHub Actions** 的免维护环境相结合，实现了从“过盾登录”到“发票生成”的全流程自动化，并且通过**智能休眠**和**自随机调度**极大地降低了风控风险。

## ✨ 核心特性

1. **🛡️ 智能绕过 Cloudflare Turnstile (过盾)**
   - 内置先进的 `TurnstileSolver`。
   - 包含多重突破手段：Shadow Root 穿透提取、模拟鼠标物理轨迹漂移点击、元素盲点偏移等方式。有效攻破 CF 盾拦截。
2. **😴 智能休眠算法 (Smart Sleep)**
   - **拒绝每日瞎跑**：告别每天凌晨启动浏览器造成的资源浪费和风控暴露。
   - **精准倒计时**：脚本会在每次运行时获取服务器的确切 `Days Left`（距到期天数）和 `Threshold`（可续期阈值）。
   - **动态设置休眠**：它将自动推算出下次需要执行的准确日期，并自动休眠相应的时长。只有在到期前需要干活的那一天，它才会醒来。
3. **🎲 Cron 自修正防检测**
   - 为了防止固定时间执行导致服务端检测规律，本项目独创了**自修改源码机制**。
   - 每次运行结束前，Python 会通过随机数（目前设定在北京时间早上 8:30 ~ 8:59 之间），生成下一个精准日期的独立 Cron 表达式，并通过 Git 命令写回工作流文件中。
4. **📡 内置 Xray 万能节点解析**
   - 支持直接传入未经处理的各种代理订阅链接（如 `vless://`, `vmess://`, `trojan://`, `ss://`, `socks5://`）。
   - 脚本会自动解析链接、下载 Xray 核心并在本地拉起透明代理环境，若有多个节点会自动轮询测速连通性。
5. **📲 完善的 Telegram 通知体系**
   - 续期成功、跳过、异常崩溃时都会发出通知。
   - 发生错误时，自动抓取浏览器实时画面（截图）随报错发送，更直观定位问题。
   - **支持 Topic 话题分类**：不仅可以发到群组，还可以指定特定的群组子话题 (Thread)。

## 📂 文件结构解析

*   **`main.py`**
    核心控制逻辑。负责启动 Chrome (配合无头 xvfb)、恢复 Cookie 实现状态持久化、解析账户列表、绕过 CF 盾并执行 UI/API 双层点击策略完成续费。运行完毕后计算 `NEXT_RUN_DAYS`。
*   **`.github/workflows/hidencloud-auto-renew.yml`**
    GitHub Actions 编排配置。提供干净稳定的 Linux 环境安装依赖。利用 `actions/cache` 进行浏览器缓存保活，完成代理构建，最后解析休眠时间改写 `cron`，实现定时随机执行。

## 🛠️ 配置与部署指南

请前往你当前仓库的 **Settings -> Secrets and variables -> Actions** 页面，依次点击 `New repository secret`，添加以下机密环境变量：

| Secret 变量名 | 必填 | 格式与说明 |
| :--- | :---: | :--- |
| **`ACCOUNTS`** | ✅ | 你的 HidenCloud 账号密码。格式：`email---password`。多个账号可以用换行符隔开。 |
| **`PROXY_NODE`** | ✅ | 你提供的节点 URI（例如：`vless://...` 或 `vmess://...`）。支持多行多节点轮询。 |
| **`TG_BOT_TOKEN`** | ✅ | Telegram 机器人的 Token（由 `@BotFather` 获取）。 |
| **`TG_CHAT_ID`** | ✅ | 接收消息的用户 ID 或群组 ID。 |
| **`TG_THREAD_ID`** | ❌ | （可选）如果你使用的是开启了话题讨论功能的群组，可填入具体的话题 ID 以防止刷屏。 |
| **`REPO_TOKEN`** | ✅ | GitHub Personal Access Token (PAT)，需具备修改 `workflow` 和 `repo` 权限，以支持脚本修改自身的定时运行代码。 |

> **提示**：为防止在云端日志泄露，请务必保证所有凭据都在 Secrets 中添加，禁止明文写在代码内！

## 🚀 工作流程简述

1. 到了指定日期（北京时间早上 8:30~8:59 的某刻），GitHub Actions 被唤醒。
2. 缓存恢复历史 Cookie，启动本地代理环境。
3. 进入 `main.py` 登录并爬取所有服务。
4. 若服务器**未到续费期** -> 计算距符合续费条件还有几天 -> 生成休眠时长。
5. 若服务器**可续费** -> 创建发票、进行结算 -> 标记休眠（成功后默认休息 6 天）。
6. 回写 `cron`，将明天的随机时间改为“N 天后”的随机时间，推送代码。
7. 发送 Telegram 通知报告并安息，等待下次唤醒。
